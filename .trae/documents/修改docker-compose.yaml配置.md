# 修改docker-compose.yaml配置计划

## 分析差异
1. **路径差异**：示例使用`./elasticsearch7/`路径，现有使用`./es/`路径
2. **网络配置**：示例使用外部网络，现有使用内部创建网络
3. **端口映射**：示例使用自定义端口，现有使用默认端口
4. **健康检查**：现有配置有详细健康检查，示例没有
5. **启动流程**：用户提到连接问题和复杂的启动流程

## 修改内容
1. **保留现有健康检查**：保持健康检查配置，确保服务就绪后再启动依赖服务
2. **调整路径**：根据示例修改配置文件路径，确保与实际目录结构一致
3. **优化网络配置**：使用外部网络配置，避免网络冲突
4. **调整端口映射**：根据示例修改端口映射，避免端口冲突
5. **添加必要配置**：添加hostname、ulimits等配置，提高稳定性
6. **优化启动流程**：确保服务启动顺序正确，避免连接拒绝问题

## 具体修改点
1. 添加`version: "3.5"`声明
2. 修改elasticsearch配置路径为`./elasticsearch7/`
3. 修改kibana配置路径为`./elasticsearch7/config/kibana.yml`
4. 修改filebeat配置路径为`./filebeat/filebeat.yml`
5. 将网络配置改为外部网络`base-env-network`
6. 调整端口映射为示例中的端口（kibana: 15601:5601, logstash: 19600:9600, 15044:5044）
7. 添加hostname配置
8. 添加ulimits配置
9. 保留现有健康检查配置
10. 优化filebeat启动依赖
